# Design: ElevenLabs Web STT API Script

Authoritative API contract lives in `research/api-contract.md` — read it first.
This doc covers architecture, data flow, contracts, and trade-offs.

## Architecture & Boundaries

Single-file Python CLI (`stt.py`) + TOML config. No internal package layering
(ponytail: fewest files). Two runtime modes via argparse subcommands:

- `stt login` — one-time browser login helper (Playwright, `channel="chrome"`
  to use the installed real Chrome — bypasses the ERR_CERT_VERIFIER_CHANGED TLS
  interception that breaks Playwright's bundled Chromium on this machine).
  Imports Playwright lazily so `transcribe` never pays the import cost.
- `stt transcribe <audio>` — pure-httpx runtime path. No browser.

Files produced:
- `stt.py` — CLI + client (auth refresh, create/poll/export).
- `config.example.toml` — documented defaults; user copies to `config.toml`.
- `requirements.txt` — `httpx`; `playwright` listed as optional (login only).
- `session.json` — generated by `login`, gitignored.
- `README.md` — usage + language reference.

## Auth Data Flow

```
stt login
  └─ Playwright(channel="chrome") opens elevenlabs.io login
     └─ user logs in (email/pw, Google, Apple, SSO — any)
     └─ read localStorage["firebase:authUser:<apiKey>:[DEFAULT]"]
        → extract stsTokenManager.refreshToken (+ localId, email)
     └─ write session.json = {apiKey, refreshToken, localId, email}

stt transcribe <audio>
  └─ load session.json
  └─ need JWT? (none cached or exp within 60s)
     └─ POST https://securetoken.googleapis.com/v1/token?key=<apiKey>
        form: grant_type=refresh_token & refresh_token=<refreshToken>
        → {id_token, refresh_token, expires_in:3600}
     └─ cache {jwt, exp=now+expires_in-60} back to session.json
  └─ httpx.Client(headers={"Authorization": f"Bearer {jwt}"})
```

Constants: `FIREBASE_API_KEY = "AIzaSyBSsRE_1Os04-bxpd5JTLIniy3UK4OqKys"`,
`API_BASE = "https://api.us.elevenlabs.io"`.
Refresh token is long-lived; JWT is 1h. Script refreshes automatically; user
only re-runs `login` when the refresh token itself expires/revoked.

## Transcribe Data Flow

```
1. load config.toml + CLI flags (flags override config)
2. validate audio file:
   - exists; size = os.path.getsize; reject if > 1000 MB (hard)
   - if ffprobe on PATH: duration = ffprobe(...); warn if > 600s (free 10-min)
     else: notice "duration unknown, ffprobe not installed"
3. (optional, --show-cost) GET /v1/speech-to-text/cost?duration=<secs>
   → print "estimated N credits"
4. POST /v1/speech-to-text/tasks  (multipart/form-data)
   fields: task_name, file, model_id="scribe_v2",
           tag_audio_events, include_subtitles,
           (language_code only if not auto),
           (no_verbatim / use_speaker_library only if true),
           (keyterms repeated once per vocab word)
   → {_id, state:"not_processing", ...}
5. poll GET /v1/speech-to-text/tasks/{_id} every ~3s
   until state=="processed" (progress 1.0, finished_at set) or error
   (print progress dots; timeout configurable, default 600s)
6. POST /v1/speech-to-text/tasks/{_id}/editor/export/{format}
   body: {"language_code": <result.language_code>}   # pass through detected/lang
   → response bytes = rendered subtitle file
   (format from config/flag, default "srt"; also txt/vtt/json/html/pdf/docx)
7. write bytes to <audio_basename>.<fmt> (or -o path)
```

Toggle defaults (script): `tag_audio_events=true`, `include_subtitles=true`
(forced on per requirement; web default is false), `no_verbatim=false`,
`use_speaker_library=false`, `language_code` omitted (=auto).

## Contracts

### config.toml
```toml
[transcribe]
language = "auto"            # "auto" | ISO 639-3 code (eng, zho, jpn, ...)
tag_audio_events = true
include_subtitles = true     # script default ON (web default OFF)
no_verbatim = false
use_speaker_library = false
vocab = []                   # e.g. ["Maj3r", "V社"]
export_format = "srt"        # srt|vtt|txt|json|html|pdf|docx
poll_timeout_secs = 600
# show_cost = false
```

### CLI surface
```
stt login
stt transcribe <audio> [-c config.toml] [--lang auto|<code>] [--events/--no-events]
        [--subs/--no-subs] [--verbatim/--no-verbatim] [--voice-lib/--no-voice-lib]
        [--vocab w1,w2,w3] [--format srt|vtt|txt|...] [-o out.srt]
        [--show-cost] [--poll-timeout 600]
stt list-languages   # print name → ISO 639-3 table (for reference)
```
Flags override config. `--vocab` merges with config `vocab`.

### session.json (generated, gitignored)
```json
{"apiKey":"AIzaSy...","refreshToken":"AMf-...","localId":"...","email":"...","jwt":null,"jwt_exp":0}
```

## Language Handling

`language` accepts `"auto"` (omit `language_code`, server auto-detects) or an
ISO 639-3 code. Confirmed: English→"eng", detected Chinese→"zho" (in
`result.language_code`). ⚠ quirk: response `user_params.language_code`
normalizes to ISO 639-1 ("en"); the REQUEST must send ISO 639-3. Script sends
ISO 639-3; export passes through `result.language_code` (which is ISO 639-3).

`stt list-languages` prints the 157-entry name→code table. The table is
extracted from the web app's JS bundle during implementation (one grep/regex
pass over the bundle) and committed as a static dict. If extraction fails,
ship a best-effort table for common languages + raw-code passthrough (users
can always pass an ISO 639-3 code directly).

## Quota / Limits

- Size: hard reject > 1000 MB before upload (stdlib `os.path.getsize`).
- Duration: best-effort via ffprobe (optional dep); warn if > 600s. No hard
  block — the real gate is server-side credits.
- Credits: free tier ≈ 10,000 credits, ≈13.9 cr/s (132.8s⇒1845 cr). Server
  rejects on insufficient credits → script surfaces the error. `--show-cost`
  calls GET /cost pre-upload to print an estimate (optional, not default).

## Dependencies

- `httpx` (runtime) — multipart upload, JSON, streaming.
- `playwright` (login only) — `pip install playwright && playwright install chrome`
  (channel="chrome" uses installed Chrome; no bundled-browser TLS issue).
- stdlib: `argparse`, `tomllib` (3.11+), `json`, `os`, `sys`, `time`, `pathlib`.
- optional: `ffprobe` (system binary) for duration pre-check.

## Trade-offs / Risks

- **HTTP replay fragility**: ElevenLabs may rename endpoints/fields. Mitigate:
  centralize endpoint URLs as constants; clear error messages on unexpected
  shapes. Accept the risk (user chose HTTP replay).
- **Refresh-token expiry**: when the Firebase refresh token expires/revoked,
  the run fails with an auth error → message "re-run `stt login`".
- **Single-file size**: `stt.py` may reach ~400 lines. Acceptable; split only
  if it grows past ~600.
- **Language map extraction**: depends on JS bundle shape; fallback is
  raw-code passthrough, so the MVP is not blocked.
- **No presigned upload**: the audio goes inline in the create POST — fine for
  ≤1000 MB (httpx streams the file). 1 GB uploads are slow but work.
