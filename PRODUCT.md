# Product

## Register

product

## Users

技术型用户与开发者——会跑命令行、有 ElevenLabs 免费账号、想把网页版「语音转文本」的免费额度榨到底的人。使用场景是在本地机器上批量转录音频（访谈、播客、长录音）导出字幕。本地 Web UI 面向同一批人：当他们不想敲一长串 CLI 参数、或需要可视化管理多账号额度池、预览切分方案时打开它。要完成的核心任务：把音频变成字幕，并在有限的免费额度内尽可能高效地分配账号。

## Product Purpose

把 `elevenlabs.io/app/speech-to-text` 的网页操作流程（登录 → 上传 → 选参数 → 等待 → 导出）压缩成一条命令或一个本地网页。复用网页内部 API 继承免费额度，通过多账号池 + best-fit 分配 + 静音切分让「12 分钟/月」的额度覆盖任意长度的音频。成功的样子：用户拖入一小时录音，工具自动切分、跨账号并发转录、合并回一份字幕，全程看得见真实进度，无需理解背后的账号调度。

## Brand Personality

工程师终端感。声音精确、克制、就事论事——像一份工整的工程笔记，而非营销文案。三词：精确、克制、功能优先（precise, restrained, functional）。界面用等宽字体标签、单色值阶、hairline 边框把命令行的可信感搬进网页；进度、日志、额度这些「机器事实」直接呈现，不加修饰。情绪目标：让用户信任它在正确地调度账号与额度，而不是被漂亮的空壳分心。

## Anti-references

- SaaS 营销风：渐变 hero、大号彩色 CTA、每个区块顶着小号全大写 eyebrow 的着陆页语法。
- 消费级 App 的圆润拟物：大圆角卡片、柔和阴影堆叠、彩色插画与吉祥物。
- 彩色仪表盘：为了「数据感」而堆砌多色图表和高饱和状态色；本工具的颜色只用于状态（绿=成功、红=失败、琥珀=警告），其余一律单色值阶。
- 玻璃拟态与装饰性动效:模糊玻璃卡、无意义的入场动画。

## Design Principles

- **机器事实直呈**:进度、额度、日志、分配计划是产品的主角,如实呈现真实的分步状态,不用假进度条或笼统 spinner 掩盖。
- **终端可信感**:等宽标签 + 单色 + hairline 边框营造命令行式的精确,让用户相信调度是对的。
- **颜色即状态**:色彩预算留给成功/失败/警告三态,其余交给 #171717→#666 的值阶;任何装饰性色彩都是噪声。
- **默认收起复杂度**:多账号手动指定、切分高级参数等专家开关默认折叠——常见路径一键完成,能力藏在「高级」后面。
- **零构建、零依赖**:前端是单文件 HTML,后端是标准库;界面的克制呼应工程栈的克制。

## Accessibility & Inclusion

基础达标(WCAG AA 级)。正文对比度 ≥4.5:1、大字/粗体 ≥3:1;注意单色值阶里靠 #666 一端的次要文字不要压到浅灰而失读。全键盘可用,焦点可见。所有动效提供 `prefers-reduced-motion: reduce` 的降级(交叉淡入或直接切换)。深浅两套主题都需各自校验对比度。
